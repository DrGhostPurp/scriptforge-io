📅 npm: date-fns
import { format, addDays } from
  'https://esm.sh/date-fns';

const today = new Date();
console.log(format(today, 'PPP'));
console.log(format(addDays(today,7),'PPP'));
🔍 npm: fuse.js (fuzzy search)
import Fuse from 'https://esm.sh/fuse.js';

const items = ['apple','apricot','banana'];
const fuse = new Fuse(items);
console.log(fuse.search('appl'));

Frequently Asked Questions

Does it support modern JavaScript (ES2023+)?
Yes. ScriptForge runs in your browser's JavaScript engine, which fully supports ES2023+ syntax including optional chaining, nullish coalescing, top-level await, dynamic imports, Array.at(), Object.hasOwn(), and more.
How do I import npm packages?
Use esm.sh as your CDN: import lodash from 'https://esm.sh/lodash'. You can specify versions: https://esm.sh/dayjs@1.11.0. Any npm package that supports ESM works this way.
Can I use async/await and fetch?
Yes. async/await works natively. Top-level await is supported. You can fetch external APIs directly — the browser's fetch API is available, and CORS-permissive APIs (like GitHub's public API) work out of the box.
How is this different from the browser console?
The browser console can't save scripts, import npm packages, share code via URL, or switch between multiple scripts. ScriptForge gives you persistent script management, npm imports, script sharing, export to .js files, and a clean UI purpose-built for scripting.
Is it completely free?
Yes. The JavaScript IDE is free forever with no rate limits. It runs entirely in your browser on GitHub Pages — there's no server for us to pay for, so there's nothing to charge for.
