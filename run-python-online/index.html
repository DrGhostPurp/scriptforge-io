How does running Python online work?

ScriptForge uses Pyodide — the official CPython interpreter compiled to WebAssembly. When you load the page, your browser downloads a ~10MB WASM binary that contains the full Python runtime. After that initial download (cached by your browser), Python runs instantly with no network calls.

This is fundamentally different from most "run Python online" services, which send your code to a server, execute it remotely, and return the output. With ScriptForge, execution is entirely local. Your code and data stay on your machine.

What Python packages are available?

Pyodide ships with over 100 packages pre-compiled to WebAssembly. The scientific Python stack is fully represented: NumPy, pandas, matplotlib, scipy, scikit-learn, statsmodels. Web-related packages like requests, BeautifulSoup, and lxml also work. Standard library modules (json, csv, re, datetime, math, statistics, itertools, etc.) are all available.
