Run Python in the Browser with WebAssembly — No Install Required

For most of web development history, Python was a server-side language. You wrote it on the server, you ran it on the server, and the browser just received the output. That constraint is now completely gone.

WebAssembly changed the rules. And Pyodide — a full CPython interpreter compiled to WASM — means you can run real Python, with real libraries, directly in the browser tab. No server, no install, no account.

How Pyodide Works

Pyodide compiles the CPython interpreter itself to WebAssembly. When you load Pyodide in a browser, you're loading a 10MB WASM binary that is, functionally, the same Python runtime you'd install on your laptop. The compilation step is a one-time cost — after the first load, it's cached by the browser.

// Load Pyodide from CDN — takes ~2-3s the first time
const pyodide = await loadPyodide();

// Run Python directly — this is real CPython
const result = pyodide.runPython(`
  x = [1, 2, 3, 4, 5]
  sum(x) * 2
`);
console.log(result); // 30

That's it. No build step, no bundler, no server request. The Python runs in the browser's main thread (or a Web Worker for non-blocking execution).

What Libraries Are Available?

Pyodide ships with over 100 packages pre-compiled to WASM, including the scientific computing stack that Python is famous for: NumPy, pandas, matplotlib, scikit-learn, SciPy, and many more. You install them the same way you would in a normal Python environment:

await pyodide.loadPackage('numpy');

pyodide.runPython(`
  import numpy as np
  arr = np.array([1, 2, 3, 4, 5])
  print(arr.mean())   # 3.0
  print(arr.std())    # 1.4142...
`);

The package download happens over CDN and is cached after the first request. For subsequent visits, your Python environment with all its dependencies loads instantly from the browser cache.

JavaScript ↔ Python Interop

One of Pyodide's most powerful features is seamless data exchange between Python and JavaScript. You can pass JavaScript objects into Python, operate on them with Python libraries, and get the result back as a JavaScript value:

// Pass JS data to Python
pyodide.globals.set('data', [10, 20, 30, 40, 50]);

const result = pyodide.runPython(`
  import numpy as np
  arr = np.array(data)
  float(arr.mean())
`);

console.log(result); // 30.0 — back in JavaScript

Performance note: Pyodide runs at roughly 1/3 the speed of native CPython for CPU-bound tasks. For most scripting, data transformation, and visualization work, this is fast enough. Web Workers let you run Python in a background thread without blocking the UI.

Real-World Use Cases

The applications are broader than they might first appear. Educational platforms use Pyodide to let students run Python exercises without any setup — the entire course runs in a browser tab. Data scientists use it to share interactive notebooks that anyone can open and run without installing Jupyter. Developer tools use it to let users write Python automation scripts that process local files entirely client-side, with zero privacy concerns.

ScriptForge uses Pyodide to power Python execution in its browser IDE. When you open a new Python script and click Run, there's no network request — your code runs in a WASM Python interpreter inside the tab itself.

Getting Started in 3 Lines

The barrier to entry is genuinely low. Drop this into any HTML file:

<script src="https://cdn.jsdelivr.net/pyodide/v0.26.0/full/pyodide.js"></script>
<script>
  loadPyodide().then(py => {
    py.runPython(`print("Hello from Python!")`);
  });
</script>

Open that in a browser, open the console, and you'll see "Hello from Python!" — with no Python installed on your machine, no server running, no build step. That's the power of WebAssembly.
