Frequently Asked Questions

Is this a real Python compiler or a simulator?
It's real. ScriptForge uses Pyodide — the full CPython 3 interpreter compiled to WebAssembly. It runs the same Python you'd install on your machine, not a simplified version or transpiler. NumPy, pandas, and other C-extension packages are also compiled to WASM and run at near-native speed.
Does my Python code get sent to a server?
No. Python execution happens entirely inside your browser tab using WebAssembly. Nothing is sent to any server. This makes ScriptForge ideal for running scripts on sensitive or proprietary data.
Which Python version does it support?
ScriptForge runs Python 3 (specifically the version bundled with the current Pyodide release, which tracks CPython closely). Python 2 is not supported as it reached end-of-life in 2020.
Can I install any Python package?
Pure Python packages can be installed from PyPI. Packages with C extensions need to be pre-compiled for WebAssembly — Pyodide ships 100+ of the most popular ones including NumPy, pandas, scipy, and scikit-learn. If a package you need isn't available, pure-Python alternatives usually work.
Is it free? Any limits?
Completely free. No rate limits, no credit card, no account required. The IDE runs in your browser and is hosted on GitHub Pages — there's no server cost for us to pass on to you.
How does it compare to Replit or Google Colab?
Replit and Colab run your code on servers — they require accounts, have rate limits, and send your code over the network. ScriptForge runs everything locally in your browser. It's faster to start, completely private, and always free. The tradeoff is that very long-running computations or filesystem-heavy work is better suited to a server environment.
